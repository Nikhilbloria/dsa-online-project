Advanced DSA Concepts Employed

This project utilizes standard and advanced computer science data structures and string-matching algorithms to deliver enterprise-grade performance directly in the client application.

🗄️ Custom HashTable

Uses a polynomial rolling hash and chaining (linked nodes) for collisions. Doubling resizing when load factor exceeds 0.75 ensures average O(1) keyword indexing.

🌳 Trie (Prefix Tree)

Indexes the joint vocabulary of the corpus. Offers O(W) word lookups, making vocabulary searches independent of the size of the corpus document collection.

🏷️ Suffix Array

Lexicographically sorts all document suffixes. Uses Binary Search to retrieve occurrences of exact phrases in logarithmic O(M log N) time.

🐎 Boyer-Moore

A pattern-matching algorithm utilizing the bad-character shift rule to bypass large swaths of text. Achieves sub-linear O(N/M) runtime averages.

🛡️ Knuth-Morris-Pratt

Applies a precomputed Longest Prefix Suffix (LPS) table to bypass backtracking in string matches. Assures strict linear O(N + M) scanning.

🧬 Rabin-Karp

Executes substring scans via sliding polynomial rolling hashes in O(N + M). Validates candidate collisions character-by-character to eliminate false matches.
